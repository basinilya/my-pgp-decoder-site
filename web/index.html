PGP URL Decoder

About PGP URL Decoder

This is a fully static, client-side tool for decrypting OpenPGP-encrypted messages that are embedded directly in a URL as a URL-safe base64 parameter (?urlsafe-pgp-message=…). Everything happens in your browser — no server ever sees your plaintext, your private key, or your passphrase.

How it differs from PrivateBin and similar paste services

This sitePrivateBin / similar
Server involvement None — fully static, can be served from GitHub Pages or any file host Requires a running server to store and serve the encrypted paste
Encryption type Asymmetric OpenPGP (RSA / ECC keypairs) Symmetric AES with a random key in the URL fragment
Where is the ciphertext? Inside the URL itself — no external storage needed On the server; the URL is only a reference to it
Key management Recipient stores their private key in browser local storage No key management — the decryption key travels with the link
Expiry / deletion The URL is self-contained; you control the link lifetime yourself Server-controlled expiry or burn-after-reading options
Who can decrypt? Only the holder of the matching private key Anyone who has the link (and the key embedded in its fragment)

The intended use case is sending a short secret to a specific person by email or chat, where the recipient already has a GPG keypair and you have their public key. The ciphertext lives entirely in the URL — there is nothing to host, expire, or delete.
